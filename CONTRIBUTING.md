# Contribuição

1. Abra uma issue descrevendo o problema ou melhoria e aguarde confirmação do escopo.
2. Crie um fork ou branch dedicado. Mantenha commits pequenos e mensagens claras.
3. Siga o estilo atual: Kotlin/Compose com nomes descritivos, funções curtas e comentários apenas onde necessário.
4. Rode `./gradlew test` e `./gradlew assembleDebug` antes de enviar; inclua testes sempre que possível.
5. Abra o pull request referenciando a issue, com resumo do que mudou e passos de teste.
